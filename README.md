# Задание 1

## Задача 1

### Текст задачи

На основе данного списка строк получить новый, где к каждой строке в конце дописан заданный символ.

### Алгоритм решения

Была описана рекурсивная функция `inputSize` для считывания размера последовательности с проверкой на корректность: принимаются только целые положительные числа, при некорректном вводе функция вызывает себя повторно. Далее описана функция `inputChar`, считывающая строку с клавиатуры: если строка пустая — возвращается пустая строка, иначе берётся только первый символ. Функция `inputUserString` формирует последовательность (`seq`) из `n` строк, считанных с клавиатуры поочерёдно, пустые строки допускаются. В `main` считывается размер через `inputSize`, затем символ через `inputChar`, после чего с помощью `Seq.map` к каждому элементу последовательности конкатенируется заданный символ. Результирующая последовательность выводится на экран.

# Задание 2

## Задача 1

### Текст задачи

Список содержит строки. Найти самую короткую строку

### Алгоритм решения

Была описана рекурсивная функция `inputSize` для считывания размера последовательности с проверкой на корректность: принимаются только целые положительные числа, при некорректном вводе функция вызывает себя повторно. Далее описана рекурсивная функция `inputSuffix`, считывающая строку-правило с клавиатуры: пустая строка не допускается, при пустом вводе функция вызывает себя повторно. Функция `generateSequence` формирует последовательность (`seq`) из `n` элементов: начиная с начальной строки `initial`, каждый следующий элемент получается конкатенацией предыдущего и строки-правила `suffix`. Далее описана функция `shortest_string`, находящая кратчайшую строку в последовательности с использованием свёртки `Seq.fold`. В качестве начального значения берётся голова последовательности, а свёртка выполняется по хвосту: в `acc` накапливается кратчайшая из просмотренных строк. В `main` считываются размер, начальная строка и строка-правило, после чего генерируется последовательность и выводится на экран вместе с кратчайшей строкой.

# Задание 3

## Задача 1

### Текст задачи

Вывести последний по алфавиту файл в указанном каталоге.

### Алгоритм решения

Описана функция `getDirectoryPath` для считывания пути к каталогу с клавиатуры. Далее описана функция `validateDirectory`, проверяющая существование каталога по указанному пути: если каталог не найден, функция завершает программу с ошибкой. Функция `getFiles` формирует отсортированную последовательность (`seq`) имён файлов в указанном каталоге: сначала получаем полные пути через `Directory.EnumerateFiles`, затем с помощью `Seq.map Path.GetFileName` преобразуем их в имена файлов, после чего сортируем по алфавиту через `Seq.sort`. В `main` считывается путь к каталогу, проверяется его существование, после чего получается последовательность файлов: если она пуста — выводится сообщение об отсутствии файлов, иначе через `Seq.last` извлекается последний по алфавиту файл и выводится на экран.
